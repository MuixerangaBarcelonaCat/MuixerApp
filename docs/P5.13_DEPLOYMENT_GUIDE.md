## Deploy a Pre-producció

### Prerequisites

1. **Backup de la base de dades de pre**
   ```bash
   # Connectar al servidor pre i fer backup
   ssh user@pre-server
   docker exec muixerapp-db pg_dump -U postgres muixerapp > backup_pre_$(date +%Y%m%d_%H%M%S).sql
   ```

2. **Git: Assegurar que tots els canvis estan commitejats**
   ```bash
   git status
   # Ha de mostrar: working tree clean
   
   git log --oneline -5
   # Verificar que el commit de P5.13 està present
   ```

### Passos de Deploy

#### 1. Push del codi a la branca de pre

```bash
# Opció A: Si pre és una branca separada
git checkout pre
git merge main  # o la branca on has fet els canvis
git push origin pre

# Opció B: Si pre utilitza tags
git tag -a v0.5.13-pre -m "P5.13 - Remove FigureFamily"
git push origin v0.5.13-pre
```

#### 2. SSH al servidor pre i pull del codi

```bash
ssh user@pre-server
cd /path/to/muixerapp

# Pull dels canvis
git pull origin pre  # o git fetch origin && git checkout v0.5.13-pre

# Instal·lar dependencies (si hi ha canvis)
pnpm install
```

#### 3. Executar la migració a pre

```bash
# Verificar que Docker està executant-se
docker ps | grep postgres

# Executar migració
cd /path/to/muixerapp
npm run migration:run

# Output esperat:
# "1 migrations are new migrations must be executed."
# "Migration RemoveFigureFamily1780982679300 has been executed successfully."
```

#### 4. Verificar migració a pre

```bash
# Connectar a la base de dades
docker exec -it muixerapp-db psql -U postgres -d muixerapp

# Executar queries de verificació (com a l'apartat "Verificació Local")
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN ('figure_families', 'figure_family_nodes');
-- Ha de retornar 0 files

\q
```

#### 5. Rebuild i restart dels serveis

```bash
# Build del backend
nx build api --configuration=production

# Build del frontend
nx build dashboard --configuration=production

# Restart dels containers Docker
docker-compose -f docker-compose.pre.yml down
docker-compose -f docker-compose.pre.yml up -d

# O si fas servir un script personalitzat
pnpm run docker:pre:restart
```

#### 6. Verificació post-deploy

```bash
# Verificar que els contenidors estan executant-se
docker ps

# Verificar logs
docker logs muixerapp-api --tail 50
docker logs muixerapp-dashboard --tail 50

# Provar l'API
curl https://pre.muixerapp.cat/api/health
curl https://pre.muixerapp.cat/api/figure-templates | jq '.data[0]'
# Verificar que no hi ha familyId ni variantOrder

# Provar el dashboard
# Navegar a https://pre.muixerapp.cat/pinyes
# Verificar les mateixes coses que a local
```

## Rollback (si cal)

Si hi ha problemes i cal fer rollback:

### 1. Rollback de la migració

```bash
# Executar rollback de la migració
npm run migration:revert

# Això revertirà la darrera migració executada
# IMPORTANT: Cal tenir una migració DOWN ben definida
```

### 2. Rollback del codi

```bash
# Tornar a la versió anterior
git checkout <commit-hash-anterior>

# Rebuild i restart
docker-compose -f docker-compose.pre.yml down
docker-compose -f docker-compose.pre.yml up -d --build
```

### 3. Restaurar backup (última opció)

```bash
# Si cal restaurar completament la base de dades
docker exec -i muixerapp-db psql -U postgres muixerapp < backup_pre_YYYYMMDD_HHMMSS.sql
```
