## Rollback (si cal)

### Opció A — Revertir la migració (només si l'API ha arrencat)

```bash
docker exec muixer-api-pre node ./node_modules/typeorm/cli.js migration:revert -d ./data-source.js
```

> **Nota:** Això recrearà les taules `figure_families` i `figure_family_nodes` BUIDES. Les dades dels nodes de família no es poden recuperar automàticament.

### Opció B — Restaurar el backup (si hi ha problemes greus)

```bash
# Aturar els contenidors
docker compose -f docker-compose.pre.yml down

# Restaurar el backup
gunzip -c ~/backups/backup-pre-p5.13-YYYYMMDD-HHMM.sql.gz | \
  docker exec -i muixer-postgres-pre psql -U muixer_pre -d muixer_pre

# Tornar a la branca anterior
git checkout <commit-anterior>

# Redeploy amb la versió anterior
docker compose -f docker-compose.pre.yml up -d --build
```

### Opció C — Tornar al codi anterior (sense restaurar BD)

```bash
# Revertir el codi
git checkout <commit-anterior>

docker compose -f docker-compose.pre.yml up -d --build

# ⚠️ IMPORTANT: Això només funciona si la migració NO s'ha executat
```
