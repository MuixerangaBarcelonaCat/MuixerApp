## Phase 1 -- Backend Event CRUD

### 1.1 CreateEventDto

Create [`apps/api/src/modules/event/dto/create-event.dto.ts`](apps/api/src/modules/event/dto/create-event.dto.ts):
- Required: `title` (IsNotEmpty, MaxLength 200), `eventType` (IsEnum), `date` (IsDateString)
- Optional: `startTime` (Matches HH:mm), `location`, `locationUrl` (IsUrl), `description`, `information`, `countsForStatistics` (default true), `seasonId` (IsUUID)

### 1.2 Expand UpdateEventDto

Modify [`apps/api/src/modules/event/dto/update-event.dto.ts`](apps/api/src/modules/event/dto/update-event.dto.ts):
- Replace current 2-field DTO with `PartialType(CreateEventDto)` from `@nestjs/swagger`

### 1.3 EventService -- new methods

Modify [`apps/api/src/modules/event/event.service.ts`](apps/api/src/modules/event/event.service.ts):
- `create(dto: CreateEventDto)`: validate seasonId if provided, create entity, return `toDetailItem(saved)`
- Expand `update()`: accept all fields from new UpdateEventDto (currently only handles countsForStatistics + seasonId). Iterate over provided fields and set them on the entity
- `remove(id)`: find event, count attendances (`this.attendanceRepository.count({ where: { event: { id } } })`), if count > 0 throw `ConflictException`, else `this.eventRepository.remove(event)`
- Note: Attendance repo needed -- inject via the existing `TypeOrmModule.forFeature` in [`event.module.ts`](apps/api/src/modules/event/event.module.ts) (Attendance already registered there)

### 1.4 EventController -- new endpoints

Modify [`apps/api/src/modules/event/event.controller.ts`](apps/api/src/modules/event/event.controller.ts):
- Add imports: `Post`, `Delete`, `HttpCode`, `HttpStatus`
- `@Post()` -> `this.eventService.create(dto)` -- returns 201
- Change `@Patch(':id')` to `@Put(':id')` (import `Put`) -- same handler, now accepts all fields
- `@Delete(':id')` + `@HttpCode(HttpStatus.NO_CONTENT)` -> `this.eventService.remove(id)`

### 1.5 Tests

Modify/create [`apps/api/src/modules/event/event.controller.spec.ts`](apps/api/src/modules/event/event.controller.spec.ts):
- POST: valid creation, missing required fields (400), invalid eventType (400), nonexistent seasonId (404)
- PUT: update all fields, partial update, 404
- DELETE: empty event (204), event with attendance (409), 404

## Phase 2 -- Backend Attendance CRUD

### 2.1 DTOs

Create [`apps/api/src/modules/event/dto/create-attendance.dto.ts`](apps/api/src/modules/event/dto/create-attendance.dto.ts):
- `personId` (IsUUID, required), `status` (IsEnum AttendanceStatus, required), `notes` (IsOptional)

Create [`apps/api/src/modules/event/dto/update-attendance.dto.ts`](apps/api/src/modules/event/dto/update-attendance.dto.ts):
- `status` (IsOptional, IsEnum), `notes` (IsOptional, nullable)

### 2.2 AttendanceService -- new methods

Modify [`apps/api/src/modules/event/attendance.service.ts`](apps/api/src/modules/event/attendance.service.ts):
- Inject `Person` repository (add `@InjectRepository(Person)`)
- `create(eventId, dto)`: validate event + person exist, create Attendance with `respondedAt = new Date()`, save, catch unique constraint error -> 409, call `recalculateSummary(eventId)`, fetch updated summary, return `{ attendance: toAttendanceItem(saved), summary }`
- `update(eventId, attendanceId, dto)`: find attendance with person+positions relations (404), update status/notes, set `respondedAt = new Date()`, save, recalculate, return `{ attendance, summary }`
- `remove(eventId, attendanceId)`: find attendance (404), remove, recalculate, return `{ summary }`
- Helper to fetch summary: `this.eventRepository.findOne({ where: { id: eventId }, select: ['attendanceSummary'] })`

### 2.3 EventController -- attendance endpoints

Add to [`apps/api/src/modules/event/event.controller.ts`](apps/api/src/modules/event/event.controller.ts):
- `@Post(':id/attendance')` -> `attendanceService.create(id, dto)` -- 201
- `@Patch(':id/attendance/:attendanceId')` -> `attendanceService.update(id, attendanceId, dto)` -- 200
- `@Delete(':id/attendance/:attendanceId')` -> `attendanceService.remove(id, attendanceId)` -- 200 (not 204, returns summary)

### 2.4 Tests

Create/modify attendance tests:
- POST: valid, person not found (404), event not found (404), duplicate (409), verify summary recalculated
- PATCH: update status, update notes, clear notes, 404
- DELETE: success, 404, verify summary recalculated

## Phase 3 -- Backend Provisional Persons

### 3.1 Person entity

Modify [`apps/api/src/modules/person/person.entity.ts`](apps/api/src/modules/person/person.entity.ts):
- Add `@Column({ type: 'boolean', default: false }) isProvisional: boolean;` after `isMember`
- TypeORM synchronize will handle the column (dev mode) or generate migration

### 3.2 CreatePersonDto -- alias validation

Modify [`apps/api/src/modules/person/dto/create-person.dto.ts`](apps/api/src/modules/person/dto/create-person.dto.ts):
- Add `@Matches(/^[^~]/)` to `alias` field to reject aliases starting with `~`

### 3.3 PersonFilterDto

Modify [`apps/api/src/modules/person/dto/person-filter.dto.ts`](apps/api/src/modules/person/dto/person-filter.dto.ts):
- Add `isProvisional?: boolean` with `@IsBoolean() @IsOptional() @Type(() => Boolean)`

### 3.4 PersonService -- provisional logic

Modify [`apps/api/src/modules/person/person.service.ts`](apps/api/src/modules/person/person.service.ts):
- `findAll()`: add `if (isProvisional !== undefined) qb.andWhere(...)` filter
- `createProvisional(alias: string)`: prepend `~`, create Person with `{ alias: "~"+alias, name: alias, firstSurname: "", isProvisional: true, isActive: true }`, catch unique constraint -> 409, return PersonResponseDto
- `update()`: handle promotion (isProvisional false -> validate name/firstSurname non-empty, alias no `~`). Handle demotion (isProvisional true -> auto-prefix `~` if needed, truncate to 20 chars)
- Add `PersonResponseDto`: add `isProvisional` field with `@Expose()`

### 3.5 PersonController

Modify [`apps/api/src/modules/person/person.controller.ts`](apps/api/src/modules/person/person.controller.ts):
- `@Post('provisional')` with a small `CreateProvisionalPersonDto` (just `alias: string, @IsNotEmpty, @MaxLength(19)`)
- Place BEFORE `@Get(':id')` to avoid route conflict (`:id` would match "provisional")

### 3.6 Tests

- POST /persons/provisional: valid, duplicate alias (409)
- PATCH promote: valid, missing name (400)
- PATCH demote: alias gets `~` prefix
- GET /persons?isProvisional=false: excludes provisionals
