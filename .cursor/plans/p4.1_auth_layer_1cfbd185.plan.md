## API Endpoints

- `POST /auth/login` вЂ” `@Public()`, LocalStrategy, returns access token + httpOnly refresh cookie
- `POST /auth/refresh` вЂ” `@Public()`, reads cookie, rotates token (reuse detection в†’ revoke family)
- `POST /auth/logout` вЂ” JWT required, revokes current token, clears cookie
- `POST /auth/logout-all` вЂ” JWT required, revokes all user tokens
- `GET /auth/me` вЂ” JWT required, returns `UserProfileDto` with linked `person`
- `POST /auth/invite/accept` вЂ” `@Public()`, activates account, auto-login (P6 hook)
- `POST /auth/setup/technical` вЂ” `@Public()` + `X-Setup-Token` header guard, bootstrap endpoint

## Key Design Notes

- **`synchronize: true` in dev** вЂ” entity changes auto-apply. `RefreshToken` entity added to `database.module.ts` entities array.
- **Production** вЂ” `isMainAccount` drop and `email NOT NULL` are safe (no existing User records per spec). For prod deploy, a manual migration or a controlled synchronize run is needed.
- **Access token** вЂ” memory only (signal), never `localStorage`.
- **Refresh token** вЂ” httpOnly cookie `muixer_rt`, stored as SHA-256 hash in DB.
- **Token family** вЂ” reuse detection revokes entire session.
- **`@Public()` decorator** вЂ” global `JwtAuthGuard` applied at `AppModule` level; opt-out per endpoint.
- **Role hierarchy** вЂ” `ADMIN(3) вЉѓ TECHNICAL(2) вЉѓ MEMBER(1)`. Dashboard routes require `TECHNICAL` minimum.
- **Rate limiting** вЂ” `@nestjs/throttler` on auth endpoints (100 req/15min per IP, per security rules).
- **Existing controllers** вЂ” `PersonController`, `EventController`, `SeasonController` get `@Roles(UserRole.TECHNICAL)` decorators after global guard is applied.
